# Bolna agent configuration

The voice, language model, and speech settings live in the Bolna dashboard, not
in this repository. This file is the source of truth for what to paste there, so
the agent configuration is reviewable alongside the code it works with.

## Why the script is in native script

`reminder_script` is generated by [lib/call-scripts/templates.ts](../lib/call-scripts/templates.ts)
in the parent's own writing system — తెలుగు, हिन्दी, தமிழ், ಕನ್ನಡ — never romanized.

Indic text-to-speech voices derive rhythm and stress from the native script. Fed
Latin transliteration such as `vesukunnara`, the voice has to guess the
pronunciation, and the result sounds flat and robotic. This is the single
biggest factor in whether a call sounds like a person.

The agent prompt must enforce the same rule, because the language model also
improvises its own sentences during the call. If the prompt does not forbid
romanization, those improvised lines arrive at the voice engine in Latin script
and undo the fix.

Medicine names stay in English, because families say brand names that way.

## Prompt

Instructions are written in English: instruction-following is most reliable in
English, and the safety rules must not be softened in translation. Everything
the agent *speaks* is in the parent's language.

```text
You are a warm, respectful medicine reminder assistant for an Indian family.
You are calling an elderly parent on behalf of their child. You are NOT the child.

## Language
- Speak ONLY in the language named in {script_language}.
- Write that language in its OWN NATIVE SCRIPT. For Telugu use తెలుగు script.
- NEVER use romanized or transliterated text (never "vesukunnara" — always "వేసుకున్నారా").
- Medicine names may stay in English, as families say them that way.

## What to say
- Begin with the exact text in {reminder_script}. Do not rewrite it.
- Greet ONCE. Ask about the medicine ONCE. Close ONCE. Then end the call.
- Keep every sentence short. Speak slowly. Wait patiently for a reply.
- If there is no reply, ask once more, then close politely.

## Never
- Never give medical advice, a diagnosis, or a dosage change.
- Never pressure, guilt, or alarm the parent.
- Never claim to be their son or daughter.
- If asked a medical question, say their family or doctor will help, and close.

## Before ending, record reminder_outcome as exactly one of:
- taken       — they confirmed taking it
- later       — they will take it later
- help        — they are unwell, in pain, or need help
- no_response — no clear answer
```

## Post-call extraction

Configure a post-call extraction field named `reminder_outcome` whose value is
one of `taken`, `later`, `help`, or `no_response`.

This is not optional. Without it the app falls back to scanning the transcript
for keywords, and a parent who phrases their answer unusually is recorded as
having missed the dose — which sends the family a false alarm. A call log whose
notes read `Bolna voice agent (transcript): ...` was classified by that fallback
and means the extraction field is missing or misconfigured.

## Context variables sent with every call

`call_log_id`, `parent_name`, `medicine_name`, `dosage_instruction`,
`scheduled_time`, `script_language`, `reminder_script`, `retry_count`.

## Voice

Sarvam is the strongest option for Telugu in Bolna; its Indic voices are built
for native-script prosody, which is why the script fix matters more than the
provider choice. If the delivery still sounds wrong after this, try a different
Sarvam speaker or a newer Bulbul model before changing provider.

## Webhook

Point the agent's webhook at:

```text
https://<your-domain>/api/webhooks/bolna?secret=<BOLNA_WEBHOOK_SECRET>
```

Bolna emits events while a call is still ringing or in progress, and can deliver
the same event twice. The route ignores in-progress events and claims the call
log before finalizing, so neither case can close a reminder early or duplicate
an alert.
